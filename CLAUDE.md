# Project: NUM Lost and Found System (МУИС-ийн гээгдсэн/олдсон эд зүйлсийн систем)

## 1. Core Principles
- **Language:** Бүх UI текст, placeholder, alert, болон хуудасны агуулга ЗӨВХӨН Монгол хэл дээр байна.
- **Design System:** Маш энгийн, цэвэрхэн (clean, modern), ашиглахад хялбар UI. Цагаан/цайвар хөх дэвсгэртэй, ойлгомжтой spacing-тэй байна.
- **Components:** Tailwind CSS болон Shadcn UI (эсвэл төстэй нээлттэй сан) ашиглана.
- **Responsiveness:** Мобайл болон дэлгэцэнд бүрэн нийцсэн (Mobile-first) байна.

## 2. Page Structures & Requirements

### 2.1 Global Layout
- **Header:** Зүүн талд Logo, баруун талд "Нэвтрэх" (Login) товч. Нэвтэрсэн үед User Profile харагдана.
- **Footer:** Зөвхөн дараах текстийг агуулна: `© 2026 3 Алхаад ол, Inc. All rights reserved.`
- **Auth Flow:** Login болон Sign up үйлдэл тусдаа хуудас руу үсрэхгүй, **Modal (Dialog)** байдлаар нээгдэнэ. Пост оруулахын тулд заавал нэвтэрсэн байх шаардлагатай (Постлох товч дарах үед шалгаад үгүй бол login sign up modal гарна).
**Бүртгүүлэх (Sign Up):** SISI ID (Оюутны код), Утасны дугаар, password, facebook account
**Нэвтрэх** Утасны дугаар, gmail, password,

### 2.2 Home Page (Нүүр хуудас)
Нүүр хуудас нь 2 үндсэн хэсгээс (section) бүрдэнэ.

**Section 1: Hero Section (Үндсэн үйлдэл)**
- Өндөр нь `< 100vh` байна (хэрэглэгч доошоо гүйлгэх/scroll хийх сэдэл өгөхүйц хэмжээтэй).
- Гол хэсэгт "Lost" (Хаясан - Улаан өнгөтэй) болон "Found" (Олсон - Ногоон өнгөтэй) гэсэн 2 том товч байна.
- Эдгээр товч дээр дарахад Зар оруулах **Modal** нээгдэнэ.

**Section 2: Feed & Filter Section (Зарын жагсаалт)**
- Дээд хэсэгтээ "Хаясан" (Lost) болон "Олсон" (Found) гэсэн 2 Tab/State товчтой байна. Энэ төлөвөөс хамаарч доорх зарууд солигдож харагдана.
- **Sidebar эсвэл Topbar:** Хайлт хийх (Search bar) болон Шүүлтүүр (Filter) байна.
- **Filter-ийн төрлүүд:** - Ангилал (Category)
  - Байршил (Location)

### 2.3 Post Creation Modal & Payment Logic
1. **Зар оруулах форм:** Зураг, Байршил, Ангилал, Огноо, Тайлбар.
2. **Олсон зар (Found):** "Баталгаажуулах асуулт" болон түүний "Зөв хариулт"-ыг оруулна.
3. **Хаясан зар (Lost):** "Шагнал санал болгох" хэсэгтэй (Мөнгөн дүн).
4. **Нэмэлт үйлчилгээ:** "Facebook групп руу түгээх" чекбокс.
5. **Төлбөрийн цонх (Payment Step):** - Хэрэв шагнал амласан эсвэл FB түгээх үйлчилгээ сонгосон бол "Нийтлэх" товч дарсны дараа шууд нийтлэгдэхгүй.
   - **QPay QR код** болон **Дансны мэдээлэл** бүхий цонх гарч ирж, мөнгө байршуулсны (Escrow) дараа зар нийтлэгдэнэ.

### 2.4 Verification Logic (Баталгаажуулах процесс)
- Хэрэглэгч "Олдсон" зар руу ороод Баталгаажуулах асуултад хариулна.
- Хэрэв хариулт зөв бол (match logic): Дэлгэц дээр олсон хүний **Утасны дугаар** болон **SISI ID/Имэйл** харагдаж, шууд холбогдох боломжтой болно.

### 2.5 User Profile & Payout (Мөнгө татах)
- **User Profile:** Өөрийн оруулсан зарууд харагдана.
- **Шагнал татах (Withdrawal):** Системд байршсан шагналтай эд зүйлийг олж өгсөн хэрэглэгчийн профайл дээр "Шагнал авах хүсэлт илгээх" товч харагдана. Хэрэглэгч өөрийн банкны дансаа оруулан мөнгөө системээс татаж авна.

### 2.6 Admin Dashboard
- Хэрэглэгчид, зарууд, төлбөрийн гүйлгээ (QPay баталгаажуулалт), болон хэрэглэгчдийн "Шагнал татах" хүсэлтүүдийг хянаж, баталгаажуулах хэсэгтэй байна.

### 2.7 Post detail 
1. **Зарын дэлгэрэнгүй хуудас:** Баталгаажуулах асуулт заавал байна .
